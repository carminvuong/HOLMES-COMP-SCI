advantages of placing style information in...

...a semantic element like h1

Placing style information in a semantic element can be useful if you want to alter only a single specified element and not all elements of the same type.

...a style element in the head

Placing style information in the head element can be useful if you want to alter all elements of the same type in the same document. This would be better than manually adding the style information to every element, as this would not violate NTTSTT, thus making your code less prone to errors.

...a separate style sheet

Placing the style information in a different style sheet would be useful if you wanted to apply the style to the all elements of the same type in different documents. Thus, this would be better than manually adding the style information to the different documents, because this would be an even better way to not violate NTTSTT.
